GIT COMMIT (Procedure) --------------------------------------------------------------------------------

1. Check Git Version 

    - Open the terminal and type

                git --version

                Output: git version 2.45.1.windows.1


2. Set up your identity

    - Write in the terminal

            git config --global user.name "Your name"
            git config --global user.email "Your email"


3. Go to github and create a new repository, give it a name, do not add README, .gitignore, or license, and lastly copy the given link

4. Git initialization 

    - In the terminal type, go inside your project patch using `cd` and lastly type `git init`

            Example:

                C:\Users\you\project\myProject> git init


5. Add all the files and folders

    - In the terminal type

        git add .  

        NOTE!!! This adds all of the folders, files, and sub-folders.

6. Commit

    - In the terminal type

        git commit -m "Your message"

7. Connect your local repo to github

    - Using the copied repo link from github type it her

        git remote add origin https//github.com/username/yourProject.git

    
    - Verify if it has connected

        git remote -v

        Output:

        origin https//github.com/username/yourProject.git
        origin https//github.com/username/yourProject.git

8. Push to github

    - In the terminal type

        git branch -M main

        git push -u origin main


EF CORE IMPLEMENTATION
    ** Always add the name convention package before initial migration

    1st - Install the EFCore package

        dotnet add package Microsoft.EntityFrameworkCore.SqlServer
        dotnet add package Microsoft.EntityFrameworkCore.Tools

        dotnet add package Pomelo.EntityFrameworkCore.MySql ** If using MySql

        dotnet add package EFCore.NamingConventions ** For table and column name translate from pascal to snake_case 

    2nd - Create your model class, put it inside the `Model` folder

        public class User
        {
            public int Id { get; set; }   // Primary key
            public string Name { get; set; }
            public string Email { get; set; }
        }

    3rd - Add a DbContext.cs file inside Data folder 

        using Microsoft.EntityFrameworkCore;

        public class AppDbContext : DbContext
        {
            public AppDbContext(DbContextOptions<AppDbContext> options)
                : base(options)
            {
            }

            public DbSet<User> Users { get; set; }
        }

    4th - Add a connection string inside appsettings.Development.json

        {
            "ConnectionStrings": {
                "DefaultConnection": "Server=localhost;Database=MyAppDb;Trusted_Connection=True;"
            }
        }

    5th - Register Ef Core in program.cs 

        builder.Services.AddDbContext<AppDbContext>(options =>
            options.UseSqlServer(
                builder.Configuration.GetConnectionString("DefaultConnection")
            )
            .UseSnakeCaseNamingConvention() // FOR SNAKE_CASE NAMING CONVENTION
            ); 

    ** The package EFCore.NamingConventions automatically change the column and table name

    dotnet ef migrations add NameNgMigration

    dotnet ef database update
